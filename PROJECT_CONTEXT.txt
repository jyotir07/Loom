================================================================================
PROJECT CONTEXT — Multi-Provider AI Demo ("Models Catalog")
================================================================================

PURPOSE
-------
A Flask web app that exposes a single prompt box plus three dropdowns
(Provider / Modality / Model) and routes each request to the right vendor's
native API. No aggregators — every provider is integrated with its own SDK or
HTTP client. The app doubles as a reference for how to wire up each provider
individually.

It also has a small admin flow that lets you add a new model (or even a whole
new provider) from the UI: it upserts a row into Postgres, patches
`models_catalog.py`, generates a starter `providers/<name>_provider.py` file
when needed, and registers it in `providers/__init__.py`.

The app is protected by a simple session-based login (username `admin`,
password `prometrix@2026`) and is mounted under the `/models_catelog/` URL
prefix (note: that path is spelled "catelog" in routes — typo kept for
backward compatibility).

TECH STACK
----------
- Backend: Python 3, Flask 3
- DB: Postgres (via psycopg2-binary) — stores the providers/models catalog
- Frontend: vanilla HTML/CSS/JS in templates/ + static/
- Vendor SDKs: openai (also used for all OpenAI-compatible providers),
  google-genai, anthropic, tencentcloud-sdk-python, plus raw `requests` for
  BFL, Ideogram, etc.

PROVIDER / MODEL CATALOG
------------------------
`models_catalog.py` is the single source of truth. Schema per model entry:

    {
        "id":      "stable id sent from the UI",
        "name":    "label shown in the dropdown",
        "model":   "real upstream model ID"  (optional; defaults to id),
        "params":  { ...extra params merged into upstream call },  (optional)
        "input_inr_per_1m":  float,   (text models — INR per 1M input tokens)
        "output_inr_per_1m": float,   (text models — INR per 1M output tokens)
        "cost_inr":          float,   (image models — INR per image)
        "free":              bool,    (optional — free-tier flag)
    }

Currently registered providers:

Text + Image:
  - openai        OpenAI                 (GPT-5/4.1/4o/o3 family, GPT Image 1/1.5, DALL-E 2/3)
  - gemini        Google Gemini          (Gemini 1.5/2/2.5/3 family, Imagen 4, Veo 2/3)

Text only:
  - anthropic     Anthropic Claude       (Haiku 4.5, Sonnet 4.6, Opus 4.7, 4.1 Opus, Mythos Preview)
  - xai           xAI (Grok)             (Grok 4.1 Fast)
  - mistral       Mistral                (Ministral 3B, Mistral Large 3)
  - deepseek      DeepSeek               (V3 chat, V4 Pro reasoner)
  - minimax       MiniMax                (M2.7)
  - zhipu         Z.AI (GLM)             (GLM-5 Reasoning)
  - perplexity    Perplexity             (sonar)
  - together      Together AI            (Llama 3.3 70B Instruct Turbo)

Image only:
  - bfl           Black Forest Labs      (Flux 2 Klein 4B/9B, Flex, Pro, Max, 1.1 Pro)
  - seedream      ByteDance Seedream     (Seedream 4.5 via Volcano Engine ARK)
  - hunyuan       Tencent Hunyuan        (Hunyuan Image 3.0)
  - ideogram      Ideogram               (Ideogram 2.0)

Video is reserved in the catalog (Gemini Veo entries exist) but not yet
implemented in any provider module.

Mistral, DeepSeek, xAI, MiniMax, Z.AI, Perplexity, Together, and Seedream all
share the OpenAI chat-completions wire shape, so they reuse
`providers/_openai_compatible.py`.

HTTP ENDPOINTS (all under /models_catelog/ except the root redirect)
--------------------------------------------------------------------
GET  /                              -> redirects to /models_catelog
GET  /models_catelog                -> main UI (login required)
GET  /models_catelog/login          -> login form
POST /models_catelog/login          -> creds check, sets session
GET  /models_catelog/logout         -> clears session
GET  /models_catelog/api/catalog    -> JSON of providers + modalities + models
GET  /models_catelog/api/diagnostic -> per-env-var {present, length,
                                       looks_like_placeholder,
                                       has_surrounding_whitespace}
GET  /models_catelog/api/code       -> source of the relevant provider helper
                                       (?provider=&modality=)
POST /models_catelog/api/generate   -> body: {provider, modality, model, prompt}
                                       returns {kind: "text"|"image", ...}
POST /models_catelog/api/models/add -> body: {provider_key, provider_label,
                                       model_id, model_name, modality}
                                       Performs: DB upsert -> patch
                                       models_catalog.py -> (if new provider)
                                       generate provider_<key>.py -> register
                                       in providers/__init__.py. Rolls back
                                       prior steps on failure.

Generate response shapes:
    text  -> {"kind": "text",  "text": "..."}
    image -> {"kind": "image", "images": [{"mime_type": "...", "data_b64": "..."}]}

Errors from upstream are surfaced verbatim in the UI.

--------------------------------------------------------------------------------

DATABASE SCHEMA (Postgres, default schema = "public")
-----------------------------------------------------
Two tables, created by seed_db.py:

    <schema>.providers (
        id, key, label
    )

    <schema>.models (
        id, provider_id, model_id, name, upstream_model, modality, params,
        input_inr_per_1m, output_inr_per_1m, cost_inr, is_free
    )

Idempotent upserts keyed on (provider key) and (provider_id, modality, model_id).

CONTRACT FOR PROVIDER MODULES
-----------------------------
Each providers/<name>_provider.py module must expose:

    def generate(modality: str, model: str, params: dict, prompt: str) -> dict

The Flask layer calls this via providers.generate(). The dict returned must
be JSON-serializable in one of the shapes above. Raise exceptions for errors;
they'll be turned into 4xx/5xx JSON by app.py:

    RuntimeError      -> 400
    NotImplementedError -> 501
    anything else     -> 500 with {"error": "ClassName: message"}

Conventions seen across the existing modules:
- A module-private `_client()` helper returns the configured SDK client.
- Per-modality helpers named `_text(...)` and `_image(...)` keep `generate`
  thin. `/api/code` introspects these by name to show their source in the UI.
- BFL and Hunyuan use async submit-then-poll APIs and time out after ~2-3 min.

--------------------------------------------------------------------------------

ADDING A NEW PROVIDER (manual path)
-----------------------------------
1. Add an entry to CATALOG in models_catalog.py.
2. Create providers/<name>_provider.py exposing
   generate(modality, model, params, prompt). For OpenAI-compatible APIs
   that's ~10 lines — see mistral_provider.py for a template.
3. Register it in providers/__init__.py (both the import and REGISTRY dict).
4. Add any new env vars to .env.example and to ENV_KEYS in app.py.

No frontend changes are required — the UI is fully driven by /api/catalog.

(The /api/models/add endpoint automates steps 1-3 for the simple cases.)

NOTES / GOTCHAS
---------------
- All keys live server-side; the browser only talks to this Flask app.
- Image responses are returned as base64 data URLs and rendered inline — no
  files written to disk.
- Upstream errors are surfaced verbatim in the UI (useful while wiring new
  models).
- Vendor model IDs change frequently. If a call returns "model not found",
  update the `model` field on the affected catalog entry; no other code
  change should be needed.
- The URL prefix is spelled "models_catelog" (typo). Don't "fix" it casually —
  any deployed bookmarks/integrations depend on it.
- The pricing fields (input_inr_per_1m / output_inr_per_1m / cost_inr) are in
  Indian Rupees per 1M tokens (text) or per image (image), based on each
  vendor's USD list price converted at a fixed rate at the time of entry.
- App is currently on branch `feat/allai`. The default branch is `main`.
